初步使用WebPack构建工具

  • webpack

Webpack 是一个很nice的脚手架，不仅可以用来打包模块，更可以用其他预处理器转成源码，比如通过配置bable，从而可以使用 ES6 。

1. 安装

Webpack运行在Node环境下，首先需要安装Node.js，自行google 。

2. Hello Word
  • 新建空目录并进入
  • 新建文件 entry.js,并写入
  • 新建文件 index.html，并写入
  • 运行

webpack 将 entry.js 打包成 bundle.js ,用浏览器打开 index.html 便会显示 Hello Word 。

3. 使用配置文件

上面的Hello word 是最简单的 webpack 使用方式。实际应用中，我们会使用配置文件。

  • 新建 webpack.config.js ,

– entry : 入口配置项

– output : 输出配置

—- path : 输出位置

—- filename : 主入口的文件名,可以增加 [hash] 从而输出带hash值的文件名，eg: bundle.[hash].js

运行

依旧生成 bundle.js ，刷新index.html 页面，显示Hello World。

  • 复杂些的配置

我们新建些 js css文件

修改 webpack.config.js 文件

– module: 模块

—- loaders: 这是webpack最强大的地方，通过 loaders,可以进行各种转换，使用 babel、autoprefixer、sass，图片压缩，tpl 转成 html 。此处将 css 文件打包进 bundle。js 。

修改 entry.js 文件

执行 webpack

发现打包失败，报错了，无法解析 style 模块。
原因是 webpack 本身只能处理 javascript，我们需要安装 css-loader 模块去处理 css 文件，也需要安装 style-loader 模块来应用 css 文件的样式。

再次执行 webpack

成功打包，刷新 index.html 页面，出现 Hello World test1 test2，并且背景变成了黄色。
